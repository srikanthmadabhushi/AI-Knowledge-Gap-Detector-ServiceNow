# -AI-Knowledge-Gap-Detector-ServiceNow-
## 📌 Overview
This project demonstrates an AI-inspired solution built on ServiceNow to detect repeated incidents and identify knowledge gaps in IT operations.

It transforms Incident Management by proactively suggesting when a Knowledge Article should be created.

---

## 💡 Problem Statement

In many organizations:
- The same incidents occur repeatedly  
- No knowledge articles are created  
- Teams resolve issues manually each time  

👉 Result: Increased workload and inefficient operations  

---

## 🎯 Solution

This project introduces an intelligent engine that:

- Detects repeated incidents  
- Identifies knowledge gaps  
- Suggests knowledge article creation  
- Improves operational efficiency  

---

## 🔥 Key Features

### 🔁 1. Incident Repetition Detection
- Tracks incidents with similar short descriptions  
- Counts occurrence patterns  

---

### 🧠 2. Knowledge Gap Detection
- Flags repeated issues as knowledge gaps  
- Uses threshold-based detection logic  

---

### 💡 3. Knowledge Suggestion Engine
Automatically suggests:
## Repeated incidents detected. Consider creating a Knowledge Article.
---

### 🖱️ 4. UI Interaction (Check Knowledge Gap Button)

Custom UI button allows users to:

- Instantly check knowledge gaps  
- View recommendations  

---

## 🔍 Example Output
Knowledge Gap Detected: True
Suggestion: Repeated incidents detected. Consider creating a Knowledge Article.

---

## 🏗️ Architecture

Incident Record
↓
Business Rule (Pattern Detection)
↓
Knowledge Gap Flag + Suggestion
↓
UI Action (Check Knowledge Gap)
↓
Client Script (Display Result)
